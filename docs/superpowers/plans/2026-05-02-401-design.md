# #401 v0.11.0 Walkthrough Fixes — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship two follow-up changes to v0.11.0 in response to issues surfaced during the live walkthrough on `findajob-test`. Split into two PRs:

- **PR A (this plan, → v0.11.1 patch):** P0 only — rip the deprecated `OPENROUTER_OPERATOR_KEY` operator-funded path, correct the per-onboarding cost claim from ~$0.50 to ~$5, and amend the v0.11.0 CHANGELOG entry. Pure mechanical revert + doc sweep.
- **PR B (separate plan, → v0.12.0 minor):** P1 — UI/UX bugs (auto-scroll, markdown rendering, missing Finalize button placeholder, premature resume banner, etc.) and prompt revisions (proactive category suggestions, numbered lists, hide `<<<FILE:>>>` blocks from chat, anti-re-emit guidance). Will be authored as its own plan when this one is done.

**Why split:** PR A is small, surgical, and unblocks the cohort bump path the fastest. PR B requires UX judgment + a fresh `findajob-test` walkthrough cycle to verify (~$5 + ~1h per cycle); bundling them would slow the cohort bump and entangle a no-judgment patch with judgment-heavy UX work. The operator already approved this split in the kickoff for #401 (see issue body, "Planning" section).

**Tester-safety contract:** alice/papa/dave/judy/tango stay on `:v0.10` for the duration of PR A. PR A only changes behavior on the `:latest` / future-`:v0.11.1` image. No compose, schema, or bind-mount changes — `migration-required: no`. The `findajob-test` stack (operator's, on `:latest`) gets the change as soon as the image publishes.

**Spec:** Issue #401 body (acceptance criteria, P0 + P1 split). The walkthrough artifacts at `tmp/onboarding-walkthrough-2026-05-02/` are local-only evidence; the issue body is canonical for what to fix.

## Goal + scope

### In scope (PR A)

**P0 — Revert the operator-funded chat path entirely.**
The operator clarified after v0.11.0 shipped that `OPENROUTER_OPERATOR_KEY` was deprecated and that path was never supposed to be supported. The v0.11.0 implementation (#400) re-enabled it via the precedence flip in `_resolved_chat_key`. PR A removes:

- `_resolved_chat_key` operator-key precedence branch in `findajob.web.routes.onboarding_interview`
- `chat_subsidized_by_operator` Jinja global registered in `findajob.web.app`
- The Step 2 cost-notice branching in `templates/onboarding/index.html`
- The lifetime-cost-badge gating in `templates/_nav.html`
- The operator-key fallback in `interview_runner.OnboardingInterviewRunner` (`operator_key` param, `_unavailable_503` messaging)
- All references to `OPENROUTER_OPERATOR_KEY` from `CLAUDE.md`, `docs/setup/configure.md`, `docs/setup/README.md`, and module docstrings in `routes/onboarding.py` and `routes/onboarding_interview.py`
- Tests covering operator-key behavior — rewrite to assert the only-tester-key invariant

**P0 — Correct the per-onboarding cost claim.**
The v0.11.0 walkthrough cost $5.39 over 100 turns / ~70 minutes with prompt caching enabled. The "~$0.50 with caching" estimate was off by a factor of ~10. Replace with **"~$3-6"** (range; the walkthrough is one data point, range gives honest envelope) across:

- `templates/onboarding/index.html` Step 2 notice
- `docs/setup/README.md`
- `docs/setup/install-docker.md`
- `README.md` (line 126 "~$1 per in-app onboarding interview" + line 155 "~$0.50 per onboarding")
- `CLAUDE.md` (line 227)
- `CHANGELOG.md` v0.11.0 entry (lines 15, 22) — amend in place, leaving the wrong claim with a strikethrough or "(corrected in v0.11.1: …)" note

**Add v0.11.1 [Unreleased] section to CHANGELOG.md** capturing both reverts.

### Out of scope (deferred to PR B / v0.12.0)

- Resume-banner gating on history length

- Any `findajob-test`-specific dogfood references that don't imply operator-funded chat (e.g. mentions of `findajob-test` as the dogfood instance for #389 NUX walkthroughs).

## Task 3: Write `docs/usage.md`

**Files:**
- Create: `docs/usage.md`

**AC from #11:** a non-technical user can identify what to do in each `/board/` tab without reading code.

**Spec outline:** §"`docs/usage.md`" (lines ~64–103 of spec).

**Sections required:**

1. **Lead paragraph + "The daily loop"** — 5-step morning routine in plain language:
   1. Check ntfy for overnight triage count
   2. Open `/board/` Dashboard — scan score ≥7 rows
   3. Flag prep-worthy jobs (STATUS → Flag for Prep) → prep runs in background
   4. Open `/materials/<company_folder>` when ntfy says prep complete → review resume + cover letter
   5. Submit applications, mark STATUS → Applied

2. **The Dashboard (`/board/dashboard`)**
   - Plain-English column meaning: relevance_score (does this job match your profile?), fit_score (how good is the actual role match?), probability_score (are you likely to pass the resume screen?)
   - STATUS dropdown options and what each triggers: Flag for Prep (launches prep subprocess), Regenerate (re-runs prep), Applied (jump straight to Applied), Waitlist (defer), Reject (remove with reason)
   - REJECT_REASON column — when and why to use each of the 11 options (source: `src/findajob/web/templates/board/_reject_cell.html`):
     1. **Too Senior** — role is a level above you
     2. **Too Junior** — role is a level below you
     3. **Skills Mismatch** — title matches but the stack / domain doesn't
     4. **Too TPM-Heavy** — ops role drifted into pure program management (rename as needed; see #65 — this string will neutralize when the scorer prompt is profile-driven)
     5. **Geography/Onsite** — unworkable location
     6. **Company Not a Fit** — ethical / culture / trajectory objection
     7. **Comp Too Low** — published band below your floor
     8. **Low Fit Score** — scorer rated low and you agree
     9. **Stale/Closed** — posting is dead
     10. **Already Applied** — duplicate of one you've already sent
     11. **Other** — free-form; paired with a note
   - `contacts` column — what the count means (populated LinkedIn connections at that company)

3. **What happens when you Flag for Prep**
   - Stage progression: `scored` → `prep_in_progress` → `materials_drafted` → (apply) → `applied`
   - Materials folder gets created under `companies/{Company}_{Title}_{date}_{time}/`
   - `/materials/<folder>/` serves the folder via the web UI (no Drive, no download needed)

4. **The Review tab (`/board/review`)**
   - What lands here: `stage=manual_review` — scorer flagged the output as uncertain
   - Promote → moves to `scored`, ready for Dashboard; Reject → standard reject flow

5. **The Applied tab (`/board/applied`)**
   - STATUS options post-application: Interviewing, Offer, Not Selected, Withdrew
   - Row color coding: green (0–6d), yellow (7–13d), red (14–20d), gray (≥21d — likely ghosted), purple (Interviewing), gold (Offer)
   - `days_since_applied` is a live formula — no re-sync needed
   - `user_notes` — free-text, 800ms debounce on save

6. **The Waitlist tab (`/board/waitlist`)**
   - `stage=waitlisted` — deferred, **not rejected** (no feedback_log write)
   - Reactivate → moves to `scored`; Reject → standard reject flow
   - Waitlist resurface: when an active application at the same company is rejected/withdrawn, ntfy notifies you to reconsider the waitlisted job

7. **The Archive tab (`/board/archive`)**
   - Every job ever ingested; pagination, sort, filter by score/stage/company/date
   - Replaces Sheet1 as the full archival view

8. **Materials viewer (`/materials/`)**
   - Files generated per prep run: `resume.md`, `cover_letter.md`, `company_briefing.md`, `network_outreach_*.md`
   - `.docx` download link for the resume (what you actually submit)
   - Links to the JD (stored from ingest)

9. **`<details>` For advanced users** (single block at bottom):
   - Stage names in the DB (`scored`, `manual_review`, `prep_in_progress`, `materials_drafted`, `applied`, `interview`, `offer`, `rejected`, `not_selected`, `waitlisted`)
   - Every STATUS dropdown action maps to a POST handler in `findajob.web.routes.board_actions` that calls into `findajob.actions`
   - Google Sheet Dashboard/Applied/Review/Waitlist tabs remain as one-way read-only mirrors until #14 ships
   - Notes written via the web also appear in the Sheet on the next `sync_sheet.py` tick

## Task 4: Write `docs/troubleshooting.md`

**Files:**
- Create: `docs/troubleshooting.md`

**AC from #11:** a user can diagnose every symptom in the index without reading source code.

**Spec outline:** §"`docs/troubleshooting.md`" (lines ~140–175 of spec).

**Sections required:**

1. **Reading the logs first**
   - `docker compose logs scheduler --tail 200 -f` — live pipeline activity
   - `state/data/logs/pipeline.jsonl` — structured events (one JSON per line)
   - `jq -r '.event' state/data/logs/pipeline.jsonl | sort -u` — list every event type ever emitted
   - Health check alerts: `docker compose exec scheduler /app/scripts/notify.py health-check` — paragraph per warning

2. **Symptom index** (each entry: one-sentence symptom → diagnosis → fix)
   - "No new jobs appearing": check pipeline_complete event in logs; RapidAPI key; jsearch_queries.txt; Gmail OAuth token age
   - "Jobs scoring 0 or not scoring at all": aichat-ng smoke test (`docker compose exec scheduler aichat-ng -- hi`); OpenRouter key + balance; Stage 2 prefilter is expected to score 5/6 for no-JD jobs (not a bug)
   - "Prep stuck in 'Prep in Progress'": watchdog resets after 60 min; check prep subprocess error in `logs/pipeline.jsonl`; Anthropic key + rate limit
   - "Sheet not updating": `sync_sheet.py` runs every 15 min; service account credentials; `config/sheet_id.txt`. (Note: Sheet1 is being retired — use `/board/archive` instead.)
   - "Ingest form (`/ingest/`) not saving jobs": hit `/healthz`; check `state/data/pipeline.db` exists and is writable; check logs for `ingest_failed` event
   - "Gmail not ingesting": token expired → `docker compose exec scheduler /app/scripts/gmail-auth`; OAuth client type must be Desktop, not TV/limited-input
   - "Container won't start": `docker compose logs scheduler`; check mount paths match `compose.yaml`; missing `state/{data,config,...}` subdirectory

3. **Health check alert reference** — one paragraph per alert `notify.py health-check` can fire (source: `scripts/notify.py::cmd_health_check`):
   - **ERROR: triage was terminated (SIGTERM)** — systemd/supercronic timeout mid-triage; check `TimeoutStartSec`
   - **WARN: pipeline_complete not seen in last 25h** — nightly triage didn't run; check scheduler + `triage.py` logs
   - **WARN: watchdog_run not seen in last 25h** — `watchdog.py` (every 10m) isn't firing
   - **WARN: sync_complete not seen in last 25h** — `sync_sheet.py` hasn't completed; Sheet won't update
   - **ERROR: N sync_sheet failure(s) in last 25h** — GCP service account rotated / sheet ID wrong / quota exceeded
   - **ERRORS: N error events in log** — any `error` / `exception` / `failed` event in the last 25h
   - **INFO: N jobs scored None (likely LLM timeout)** — OpenRouter flaky / aichat-ng hung
   - **WARN: N source(s) returned 0 jobs despite producing jobs in last 7d — likely silent feed failure** — source names: greenhouse, ashby, lever, jobsapi, gmail
   - **WARN: low memory — N MB available** — container memory pressure
   - **WARN: high swap usage** — swap >50% utilized
   - **WARN: Sheet1 has ~N rows (threshold: SHEET1_ROW_WARN)** — Sheet1 growth past threshold (retires with #136)
   - **WARN: N null-score jobs in manual_review (scorer failure)** — scorer crashed mid-batch; aichat-ng not reachable
   - **WARN: N real-flag jobs in manual_review backlog** — scorer is flagging too much; tune profile
   - **WARN: N target-company jobs scored 3–6 in last N days (potential mis-scores)** — scorer missing obvious targets; review and add to profile Tier 1 list

4. **`<details>` For advanced users**:
   - Reading `audit_log` in `pipeline.db`: `sqlite3 data/pipeline.db "SELECT * FROM audit_log ORDER BY timestamp DESC LIMIT 20"`
   - Manually triggering resync: `docker compose exec scheduler /app/scripts/sync_sheet.py`
   - Manually re-triaging: `docker compose exec scheduler /app/scripts/triage.py --force`
